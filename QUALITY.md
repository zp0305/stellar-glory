## 二、题库内容质量标准（v2.0 混合题型）

### 2.1 题量标准

| 模型范围 | 题量/模型 |
|----------|:---------:|
| M01–M10（运动学/力学基础） | 8道 |
| M11–M26（电磁学/能量等） | 10道 |
| M27–M42（光学/近代物理等） | 12道 |

### 2.2 题型结构

| 难度 | 题型分布 | 自动判分 | 提示率建议 |
|------|:--------:|:--------:|:---------:|
| B层 | 选择×2 + 填空×1（数值） | ✅（标准化比对） | 30% |
| J层 | 选择×2 + 计算×1 | 选择题✅ / 计算题❌ | 50% |
| T层 | 选择×1 + 计算×1 | 选择题✅ / 计算题❌ | 80% |

> **填空/计算题答案比对规则**：`normalizeAnswer()` 统一处理（去 $、去 \、去空格、转小写）后比对。计算题提交后显示参考答案，学生自行对比，不自动判分。

### 2.3 题干（question）

- LaTeX 公式用 `$...$`（行内）或 `$$...$$`（独立公式块）
- 每道题必须包含完整的物理情境
- 选择题长度 1-3行，填空/计算题 2-5行

### 2.4 提示（hint）

- 每个提示只解决一个卡点，给出方向，不直接给答案
- 选择题/填空题必须有提示；计算题建议有提示

### 2.5 解析（explanation）

必须包含：① 逐一步骤说明（带公式）② 最终答案 ③ 技巧说明（如有）

### 2.6 标签（tags）

每题 1-3个，从标签库选取：`公式应用` `易错点` `真实情境` `多过程分析` `追及相遇` `综合应用` `逆向思维` `技巧性强` `概念辨析` `图像识别` `临界条件` `多物体分析`

## 三、Supabase 管理规范

### 3.1 当前状态

| 阶段 | 环境 | Supabase |
|------|------|:--------:|
| 调试/开发阶段 | 本地 .env 缺失 | ❌ 断开（isSupabaseConfigured=false） |
| 正式部署阶段 | .env 配置完成 | ✅ 启用（匿名模式） |

**当前：调试阶段，Supabase 断开。**

### 3.2 启用流程（正式部署时）

1. 在 Supabase 项目后台确认匿名登录已开启
2. 配置 `.env`：
   ```
   VITE_SUPABASE_URL=https://your-project.supabase.co
   VITE_SUPABASE_ANON_KEY=your-anon-key
   ```
3. 构建部署，Supabase 自动生效（`isSupabaseConfigured=true`）
4. 收藏夹/学习进度 数据从 localStorage 迁移至 Supabase 云端

### 3.3 数据保护原则

```ts
// 所有 Supabase 调用必须先检查
if (!isSupabaseConfigured) {
  // 安静降级 → localStorage
}
```

| Store | 调试阶段 | 正式部署 |
|-------|:--------:|:--------:|
| authStore | local session | Supabase session |
| favoritesStore | localStorage | Supabase |
| userStore（理解度）| localStorage | Supabase |
