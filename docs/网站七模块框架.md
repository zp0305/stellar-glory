# 网站七模块框架

## 总体架构

星耀一期七模块，围绕"知识元→解题套路→内化"学习闭环设计：

```
┌─────────────────────────────────────────────────┐
│  知识元（六段式）  →  公式  →  检测  →  进度     │
│        ↓                            ↺          │
│     图谱（Q3）                   自我驱动       │
│        ↓                                        │
│     题库  ←  视频（Q3）←  导航                  │
└─────────────────────────────────────────────────┘
```

---

## 模块一：知识元（六段式）

**位置**：左侧栏 → 知识元

**结构**：每个知识元按六段式展开：

1. **引入** — 从生活/实验现象切入，激发兴趣（1-2段）
2. **是什么** — 核心概念定义，简洁精准
3. **怎么用** — 适用场景、公式、注意事项
4. **公式** — 独立模块，附推导过程
5. **纵横联系** — 与其他知识元的关联（跨章节/跨学科）
6. **检测** — 费曼学习法验证（不是做题）

**导航**：章节树 → 主题 → 知识元三级结构

**状态**：✅ 物理科目已验证，内容规范 v3.0

## 模块二：公式卡片

**位置**：左侧栏 → 公式库

**内容**：全学科公式汇总，含：
- 公式本身
- 适用条件
- 推导过程（关键步骤）
- 常见变形
- 与其他公式的关系

**交互**：点击公式展开详情，支持打印

**一期**：仅物理，其他科目延后

**功能**：知识元之间的关联可视化，帮助学生形成认知地图

## 模块四：自我检测（费曼学习法）

**位置**：知识元底部 → 检测入口，或独立入口

**核心机制**（费曼学习法）：

```
第1步：选择知识元
       ↓
第2步：假想教授 — "如果我要把这个知识教给一个初一学生，
         我会怎么说？"
       ↓
第3步：发现缺口 — 哪里讲不清楚？哪里有逻辑跳跃？
       ↓
第4步：回炉重学 — 返回知识元，查漏补缺
       ↓
第5步：简化表达 — 用最朴素的语言重新表述核心概念
```

**产品形态**：

- 每个知识元底部设置「费曼检测」按钮
- 点击后展开引导式提问，3-5个问题，引导学生自我复述
- 完成后记录检测状态（已通过/需重学）

**示例（物理·加速度）**：

> 1. "不用公式，你怎么跟同学解释什么叫加速度？"
> 2. "加速度的单位 m/s² 到底是什么意思？"
> 3. "汽车提速和刹车，哪个过程加速度更大？"
> 4. "加速度为零时，物体一定静止吗？"

**设计原则**：
- 不打分，不排名，纯自我驱动
- 问题要触发"说清楚"而非"做对题"
- 一期仅物理科目，后续扩展

## 模块五：进度追踪

**位置**：个人中心 → 学习进度

**内容**：
- 章节完成度（百分比）
- 知识元完成状态（未学/已学/已检测）
- 学习时长统计
- 薄弱点提示（基于检测结果）

**交互**：可视化进度条，支持导出/打印

---

## 模块六：题库（Q2 首批内容）

**位置**：左侧栏 → 题库

**内容**：与知识元绑定的练习题，五维标签体系：
- 难度 D1-D6
- 题型：选择/填空/计算/实验/证明
- 知识点：精确到知识元
- 能力层级：理解/应用/分析/综合
- 年份/来源

**一期**：仅物理力学部分，30-50 题验证

## 导航结构

```
顶部栏
├── logo + 学科切换（物理/化学/生物/数学/英语/语文）
├── 搜索（知识元/公式/题目）
└── 个人中心

左侧栏
├── 首页（仪表盘）
├── 知识元（章节树）
├── 公式库
├── 图谱（Q3）
├── 题库

├── 返回顶部
├── 打印本页
└── 费曼检测（当前知识元）
```
