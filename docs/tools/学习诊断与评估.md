# 星耀高中教育平台 — 学习诊断与评估

> **定位**：面向六科的分层诊断体系与学习路径推荐，交互式工具页面
> **网站对接**：独立一级入口 `/diagnosis`，同时嵌入学科页和首页
> **用户操作**：选学科+选诊断类型 → 答题 → 查看层级分布+模块热力图+推荐动作
> **关联**：《内容生成总纲 v9.1》文件03
> **标准遵循**：本文件遵循《内容生成总纲 v9.1》第三章目标用户与三层能力模型、第五章内容生产标准（含题库架构v1.2的分层逻辑）、第七章质量审核门禁。

## 一、设计理念

### 1.1 为什么需要诊断系统

"不知道自己哪里不会"是高中学习最隐蔽的失分原因。

年级前10%-15%的学生中，大量人的问题不是"不会"，而是**不知道自己哪里会、哪里不会**——时间均匀分配到所有内容上，实际80%的提分空间集中在20%的薄弱模块。

| 有诊断 | 无诊断 |
|:---|:---|
| 精确定位薄弱模块，靶向突破 | 感觉"哪里都差"，均匀用力 |
| 知道自己当前在L1/L2/L3哪一层 | 不清楚能力层级，学习无层次 |
| 每次诊断后看到进步曲线 | 进步靠"感觉"，容易气馁 |
| 学习路径有数据支撑 | 路径靠经验猜测 |

### 1.2 核心原则

1. **分层诊断**：不只判"对不对"，还要判"你在哪一层"——L1/L2/L3三层能力模型（总纲§3.2）是诊断的核心框架
2. **模块定位**：不只判"整体水平"，还要判"哪个板块薄弱"——精确到知识网络中的认知板块
3. **可行动**：诊断结果必须直接推荐下一步学什么、做什么题，不做"测完就完"的纯评估
4. **轻量高效**：单次快速诊断设计目标30分钟（多数学生可完成），上限40分钟（L3题可跳过）
5. **可追踪**：历史诊断记录可对比，看到每个模块的能力增长曲线

### 1.3 与其他文件的边界

| 文件 | 边界 |
|:---|:---|
| 01学习目标与三年规划 | 01定义"阶段目标"，本文件定义"检测是否达成"——诊断标准对应01的阶段检测清单 |
| 02学习方法体系 | 02定义"怎么学"，本文件定义"怎么检测学习效果"——错题分析（02§5）为本文件提供诊断数据源 |
| 04-09六科思维框架 | 六科文件提供具体的知识点/模型/套路内容，本文件定义诊断这些内容的通用标准和方法 |
| 10跨学科知识网络 | 本文件不做跨学科诊断（V2阶段），仅做单科内模块级诊断 |
| 11知识向能力转化 | 11定义L1→L2→L3的跨越训练方案，本文件定义"学生当前在哪一层"——两者互补 |

## 二、三层诊断标准

### 2.1 能力层级定义（总纲§3.2映射）

| 层级 | 名称 | 行为特征 | 对应题库 | 正确率阈值 | 时间约束 |
|:---|:---|:---|:---|:---|:---|
| **L1** | 知识掌握 | 能复述概念、套用公式解标准题 | B级（基础） | ≥90% | 单题≤5分钟 |
| **L2** | 思维运用 | 能识别题型、选择策略、完成多步推理 | J级（进阶） | ≥60% | 单题≤15分钟 |
| **L3** | 迁移创新 | 能在陌生情境中识别底层知识结构、跨模型综合 | T级（拓展） | ≥40% | 单题≤20分钟 |

### 2.2 层级判定规则

**单题判定**：根据题目的`difficulty`和`level`标签自动对应层级——
- `level: "L1"` + `difficulty: 1-2` → L1诊断题
- `level: "L2"` + `difficulty: 3-4` → L2诊断题
- `level: "L3"` + `difficulty: 5-6` → L3诊断题

**模块判定**：一个认知板块内，某层级题目正确率达到阈值即判定为"通过"——

| 层级 | 题量（每模块） | 通过阈值 | 状态标签 |
|:---|:---|:---|:---|
| L1 | 2题 | 2/2 全对 | ✅ 通过 / ❌ 未通过 |
| L2 | 3题 | ≥2/3 | ✅ 通过 / ⚠️ 临界（1/3） / ❌ 未通过（0/3） |
| L3 | 2题 | ≥1/2 | ✅ 通过 / ❌ 未通过 |

**综合判定**（整科）：三个层级的通过情况组合——

| L1 | L2 | L3 | 综合评级 | 推荐动作 |
|:---|:---|:---|:---|:---|
| ✅ | ✅ | ✅ | 🏆 全面掌握 | 进入下一阶段学习（参照01三年规划） |
| ✅ | ✅ | ❌ | ⭐ 熟练 | 适量挑战L3，不勉强 |
| ✅ | ⚠️ | ❌ | 📈 发展中 | 优先巩固L2薄弱模块 |
| ✅ | ❌ | ❌ | 📚 基础牢固 | 集中攻L2套路 |
| ❌ | — | — | 🔧 基础待补 | 回归L1知识点，补基础 |

### 2.3 自评诊断（无需答题）

除答题式诊断外，提供一份自评清单，供学生在日常学习中快速自检——

| 层级 | 自评问题（选"是"≥3个即视为通过） |
|:---|:---|
| L1 | 1. 闭卷能默写核心公式/概念？<br>2. 看到标准题能想到第一步？<br>3. 独立完成课后基础题无障碍？<br>4. 能用自己的话解释概念？<br>5. 作业正确率稳定在85%以上？ |
| L2 | 1. 看到新题能快速识别属于哪个模型/题型？<br>2. 能选择合适的解题策略（不靠试错）？<br>3. 多步推导过程中不卡壳？<br>4. 同一模型的变式题能举一反三？<br>5. 章节综合题能独立完成60%以上？ |
| L3 | 1. 遇到从没见过的题型能找到突破口？<br>2. 能把两个以上模型/方法组合解题？<br>3. 能跨章节调动知识解决综合题？<br>4. 做错后能自己分析出深层原因？<br>5. 高考压轴题至少能写出前两问？ |

## 三、诊断卷设计

### 3.1 诊断卷结构

**快速诊断（推荐首次使用）**：

| 层级 | 题量 | 题型 | 题目来源 | 预计时间 |
|:---|:---|:---|:---|:---|
| L1 | 6题 | 选择/填空 | 各模块各1题，覆盖核心概念（近代物理与热学光学合并出题） | 10分钟 |
| L2 | 8题 | 选择/计算/填空 | 重点模型各1-2题，考查策略选择 | 15分钟 |
| L3 | 6题 | 计算/综合/实验 | 跨模块综合题，考查迁移能力 | 15分钟 |
| **合计** | **20题** | — | — | **~40分钟** |

> **实际控制**：多数学生能在30分钟内完成。L3题如果5分钟内毫无思路可跳过（计入"未通过"），不强制答题。

**模块诊断（精准定位）**：

选择特定认知板块后，仅出该板块的诊断题——

| 层级 | 题量 | 预计时间 |
|:---|:---|:---|
| L1 | 2题 | 5分钟 |
| L2 | 3题 | 10分钟 |
| L3 | 2题 | 10分钟 |
| **合计** | **7题** | **~25分钟** |

### 3.2 出题规则

**L1诊断题要求**（来源：题库v1.2的`function: "DIAG"`题目）：
- 考查单一概念或公式
- 不涉及多步推理
- 侧重概念辨析（易混点、适用条件判断）
- 错误选项需有迷惑性（常见错误路径）

**L2诊断题要求**（来源：题库v1.2的`difficulty: 3-4`题目）：
- 考查模型识别+策略选择
- 涉及2-3步推理链
- 题目场景可以是标准情境的变式
- 解析中需标注"识别了哪个模型、用了什么策略"

**L3诊断题要求**（来源：题库v1.2的`difficulty: 5-6`题目）：
- 涉及跨模型或跨板块综合
- 情境需要一定的信息提取和建模能力
- 不超纲，但在教材框架内做深度组合
- 可包含开放性设问

### 3.3 诊断卷的学科差异

| 学科 | L1侧重 | L2侧重 | L3侧重 |
|:---|:---|:---|:---|
| **物理** | 概念辨析+公式适用条件 | 模型识别+过程分析 | 跨模型综合+建模能力 |
| **数学** | 定理条件判断+基本运算 | 问题类型识别+多步推导 | 多知识综合+构造/证明 |
| **化学** | 方程式书写+实验现象 | 物质推断+反应原理 | 实验设计+信息迁移 |
| **生物** | 概念辨析+图表解读 | 知识网络串联+实验分析 | 信息提取+综合应用 |
| **语文** | 文本理解+基础知识 | 答题规范+思路组织 | 综合鉴赏+思辨表达 |
| **英语** | 语法辨析+词汇运用 | 语篇分析+长难句 | 写作/翻译+跨文化理解 |

## 四、结果页与路径推荐

### 4.1 结果页结构

诊断完成后展示三层结果：

**第一层：整体评级（总览卡片）**

```
┌─────────────────────────────────────────┐
│  你的物理水平：⭐ 熟练                    │
│                                         │
│  L1 知识掌握 ████████░░ 100%  ✅ 已通过    │
│  L2 思维运用 ███████░░░  75%  ✅ 已通过    │
│  L3 迁移创新 ███░░░░░░░  25%  ❌ 待突破    │
│                                         │
│  完成时间：12分30秒                        │
│  较上次诊断：L2 +15% ↑                    │
└─────────────────────────────────────────┘
```

**第二层：模块热力图（精准定位）**

```
各模块诊断结果：
├── 运动学      L1✅ L2✅ L3✅  → 稳固
├── 力学        L1✅ L2✅ L3❌  → L3待突破
├── 曲线运动    L1✅ L2⚠️ L3❌  → 优先巩固L2
├── 能量与动量  L1✅ L2✅ L3❌  → L3待突破
├── 电磁学      L1⚠️ L2❌ L3❌  → ⚡ 优先补L1！
├── 热学光学    L1✅ L2✅ L3❌  → L3待突破
└── 近代物理    L1✅ L2✅ L3✅  → 稳固
```

**第三层：推荐动作（可执行）**

基于诊断结果的精准推荐——

| 情况 | 推荐动作 | 对接网站功能 |
|:---|:---|:---|
| 某模块L1未通过 | 回到该模块的知识点详解，重新学习 | 模型详解页·知识Tab |
| 某模块L2未通过 | 学习该模块的核心套路（对应模型） | 模型详解页·套路Tab |
| 某模块L2临界 | 做该模块的L2练习题巩固 | 练习中心·模型练习 |
| L2已通过但L3未通过 | 尝试L3综合题，每周1-2道 | 练习中心·综合练习 |
| 全科L1/L2已通过 | 进入下一阶段（参照01三年规划） | 三年规划页·阶段检测 |

### 4.2 路径推荐算法

推荐逻辑（确定性规则，不涉及AI）：

```
输入：诊断结果（每模块的L1/L2/L3状态）
输出：推荐动作列表（优先级排序）

规则：
1. 如果任何模块L1未通过 → 置顶推荐该模块的L1内容
2. 如果任何模块L2未通过 → 其次推荐该模块的L2套路
3. 如果所有L1通过且L2通过率<50% → 推荐集中攻L2
4. 如果所有L1通过且L2通过率≥50% → 推荐尝试L3
5. 推荐内容附带具体的模型ID/套路ID/题目ID，点击直达
6. 如果学生同时拥有答题诊断和自评诊断记录，路径推荐以答题诊断为准，自评诊断仅用于补充确认
```

### 4.3 诊断频率建议

| 诊断类型 | 建议频率 | 触发场景 |
|:---|:---|:---|
| 快速诊断 | 每月1次 | 月度复盘、阶段切换时 |
| 模块诊断 | 学完一个模块后 | 章节测试后、考前冲刺前 |
| 自评诊断 | 随时 | 日常学习间隙，1分钟完成 |

## 五、历史追踪与进步曲线

### 5.1 追踪维度

每次诊断结果存储后，可生成以下追踪视图：

| 视图 | 内容 | 时间跨度 |
|:---|:---|:---|
| 能力雷达图 | L1/L2/L3三维 + 各模块维度 | 最近3次诊断对比 |
| 模块热力图 | 各模块×各层级的状态矩阵 | 最近6次诊断对比 |
| 进步曲线 | 整体正确率随时间的变化 | 全部历史记录 |
| 薄弱模块排行 | 错误率最高的模块排序 | 最近3次诊断加权 |

### 5.2 数据存储

| 字段 | 说明 |
|:---|:---|
| `user_id` | 用户标识 |
| `diagnosis_id` | 诊断记录唯一ID |
| `subject` | 学科枚举（PHY/CHE/MAT/BIO/CHN/ENG） |
| `diagnosis_type` | quick（快速）/ module（模块） |
| `target_module` | 模块诊断时的目标板块ID，快速诊断时为null |
| `timestamp` | 完成时间 |
| `duration_seconds` | 答题用时 |
| `results` | 各题结果（题号、正确/错误、用时） |
| `level_scores` | {L1: 0.83, L2: 0.67, L3: 0.33} |
| `module_scores` | {mechanics: {L1:1, L2:0.67, L3:0}, electromagnetics: {…}} |
| `overall_rating` | 综合评级（枚举） |
| `recommendations` | 推荐动作列表 |

## 六、网站交互设计

### 6.1 页面结构

```
/diagnosis
├── 入口选择页
│   ├── 学科选择（六科图标卡片）
│   ├── 诊断类型（快速诊断 / 模块诊断 / 自评诊断）
│   └── 历史记录入口（上次诊断结果摘要）
│
├── 答题页
│   ├── 进度条（当前题号/总题数 + 预计剩余时间）
│   ├── 题目卡片（题干 + 选项/填空框 + 计算草稿区）
│   ├── 上一题/下一题/提交 按钮
│   └── L1/L2/L3层级标签（让学生知道当前题目考查什么层级）
│
├── 结果页
│   ├── 整体评级卡片（评级 + 三层进度条 + 用时）
│   ├── 模块热力图（各模块×各层级状态矩阵）
│   ├── 推荐动作列表（点击直达对应内容）
│   ├── 答题回顾（逐题查看：正确答案 + 我的答案 + 解析）
│   └── "分享诊断结果" / "开始推荐学习" 按钮
│
└── 历史记录页
    ├── 诊断时间线（最近N次诊断摘要）
    ├── 能力雷达图（最近3次对比）
    ├── 模块热力图（最近6次对比）
    └── 进步曲线（正确率随时间变化）
```

### 6.2 交互要点

1. **答题页**：L3题提供"跳过"按钮，标注"跳过不计入用时，但视为未通过"
2. **结果页**：模块热力图使用颜色编码（绿=通过/黄=临界/红=未通过/灰=未测试）
3. **推荐动作**：每条推荐附带"开始学习"按钮，点击跳转到对应的模型页/套路页/题库
4. **历史记录**：默认展示最近3次诊断的对比雷达图，展开可查看全部历史
5. **自评诊断**：全屏展示自评问题，勾选式，即时出结果，无需提交
