> **基于**：星途内容生产标准 v3.2 + 六科知识网络（物v2.4/化v1.4/生v1.4/数v3.2/英v1.5/语v1.2）

## 一、体系定位

### 1.1 解决什么问题

中国高中教育的"中段塌陷"：年级前10%-15%的学生有冲985的潜质，但卡在两个陷阱里——
- **知识碎片化**：会做题，但知识点之间没有联系
- **思维固化**：遇到新题型、创新题就卡壳，不会迁移

**本体系的目标**：帮这类学生从"做题机器"跃迁到"思维型学习者"。

### 1.2 核心理念

> 不只给公式，给"怎么想"。不只给答案，给"思考的方法"。

**与星耀网站的关系**：
- 本体系是"内容生产蓝图"
- 星耀网站是"内容消费终端"
- 所有内容最终要转化为网站可渲染的JSON数据

## 二、三层架构

```
┌─────────────────────────────────────────────┐
│  第一层：内容生成总纲（本文件）                │
│  定义：目标用户、三层模型、内容标准、文件边界、  │
│        生产顺序、质量标准、审核门禁              │
├─────────────────────────────────────────────┤
│  第二层：学科内容文件（24个）                  │
│  按本文件定义的标准和边界，撰写具体内容          │
│  输出格式：Markdown（供人阅读）+ JSON（供网站）  │
├─────────────────────────────────────────────┤
│  第三层：网站数据与技术平台                    │
│  将第二层内容"翻译"为前端可渲染的数据结构        │
│  输出格式：JSON、数据库、前端组件              │
└─────────────────────────────────────────────┘
```

**关键原则**：
- 第一层**不越界**：不写具体知识点、不展开学科思维训练案例
- 第二层**不重复**：不重新定义三层模型、不重复标准，只按标准"填空"
- 第三层**不臆造**：只实现第一层和第二层明确定义的功能，不自行增加教育理论功能

## 三、目标用户与三层能力模型

### 3.1 目标用户画像

| 维度 | 描述 |
|------|------|
| 学业水平 | 年级前10%-15%，总分在985线附近或以上 |
| 核心特征 | 有自学能力，但缺乏系统性思维训练 |
| 主要痛点 | 知识碎片化、遇到新题型卡壳、不会迁移 |
| 时间约束 | 课业繁重，每天课外学习时间有限（1-2小时） |
| 使用场景 | 课前预习、课后巩固、考前冲刺、薄弱点突破 |

### 3.2 三层能力模型（贯穿全体系的核心框架）

| 层级 | 名称 | 行为特征 | 对应题库 | 占比 |
|------|------|---------|---------|------|
| **L1** | 知识掌握 | 能复述概念、套用公式解标准题 | B级（基础） | 40% |
| **L2** | 思维运用 | 能识别题型、选择策略、多步推理 | J级（进阶） | 45% |
| **L3** | 迁移创新 | 能在陌生情境中运用、跨学科迁移 | T级（拓展） | 15% |

**诊断标准**：
- L1：正确率≥90%，5分钟内完成基础题
- L2：正确率≥60%，能识别模型并完成2-3步推理
- L3：正确率≥40%，能在全新情境中识别底层知识结构

**为何是三层**：
- 可直接映射网站B/J/T分级
- 边界清晰，学生可自我诊断
- 教育学研究表明3层级评估信度高于4层级

## 五、内容生产标准（核心规范）

> **所有24个文件、所有网站JSON数据，必须遵循以下标准。**
> **权威基线**：本标准与《星途内容生产标准 v3.2》保持一致。如有冲突，以 v3.2 为准。

### 5.1 知识点内容标准（六段式）

> 来源：星途内容生产标准 v3.2 §1.1。以下为网站前端渲染的标准段落，所有知识点JSON必须按此结构组织。

每个知识点详解必须包含以下六段，**顺序固定**：

| 段落 | 名称 | 功能 | 字数要求 |
|------|------|------|---------|
| 1 | **情境锚定** | 用真实生活情境或物理直觉"锚定"学生的注意力，激活已有经验 | 100-150字 |
| 2 | **困惑预设** | 提出学生在此知识点上的常见错误/直觉误区，制造认知冲突 | 150-200字 |
| 3 | **实验/现象呈现** | 呈现实验、现象或数据，让学生"看到"问题的存在 | 200-250字 |
| 4 | **概念自然涌现** | 从情境和实验中自然引出概念，不从课本定义开始 | 150-200字 |
| 5 | **公式推导叙事** | 展示"怎么想"的推导过程，用叙事方式呈现而非干巴巴的步骤罗列 | 300-400字 |
| 6 | **迁移与应用** | 跨情境、跨模块、跨学科的应用案例，回到生活/科技场景 | 150-200字 |

**附加结构**（六段之外，知识点页面必须包含）：

| 模块 | 功能 | 说明 |
|------|------|------|
| 前置知识检测 | 3道快速检测题 | 学习本知识点前必须掌握的前置内容检查 |
| 分层变形 | 基础/进阶/挑战三层实例 | 基础变形（一步+2例）、进阶变形（多步+1例）、挑战变形（比例关系+3例） |
| 理解度自评 | A/B/C三级自评 | 帮助学生判断自己是否真正掌握 |

**禁止事项**：
- 禁止直接从课本定义开始写（必须从情境或现象出发）
- 禁止只给结论不给推导（公式推导叙事是核心段落）
- 禁止六段缺失或顺序打乱
- 禁止用空洞修辞代替实质推导（如"不难发现""显而易见"）

### 5.2 解题套路内容标准

每个解题套路必须包含：

| 要素 | 要求 |
|------|------|
| **适用场景** | 明确"看到什么特征时用这个套路" |
| **思维方法标注** | 必须标注关联的学科核心思维（见§5.4） |
| **操作步骤** | 3-7个具体步骤，可执行、可检查 |
| **典型例题** | 1道完整例题（含详细解析） |
| **变式训练** | 2-3道变式题（换条件、换情境） |
| **常见错误** | 学生用此套路时的典型失误 |
| **难度分级** | 标注L1/L2/L3 |

### 5.3 题库分级标准

| 层级 | 判定标准 | 题目特征 | 占题库比例 |
|------|---------|---------|-----------|
| **B级** | 学生只需识别知识点+套用公式/定理即可解决 | 标准题型、单一知识点、条件直接 | 40% |
| **J级** | 学生需要识别模型、选择方法、完成多步推理 | 综合题、多知识点组合、需要分析过程 | 45% |
| **T级** | 学生需要在陌生情境中识别底层结构，或跨学科迁移 | 创新题、情境题、开放性问题、跨学科题 | 15% |

> **与三层能力模型的映射**：B级对应 L1（知识掌握），J级对应 L2（思维运用），T级对应 L3（迁移创新）。在题库架构 v1.2 中，B/J/T 同属 L2 能力层级（"模型题"），但此处按能力层级划分是概念映射，两者不矛盾。

**出题自检三问**：
1. 学生"没见过同类题"时，能否用已学思维方法解决？（L2/L3必须满足）
2. 换几个数字是否变成另一道题？（如果是，只是L1变式）
3. 是否需要跨模块知识？（需要→L2/L3；不需要→L1）

## 六、内容风格指南

> **所有内容创作者（包括AI）必须遵守以下风格规范。**

### 6.1 总体基调

**定位**：学术严谨 + 语言生动，面向有自学能力的优秀高中生。

| 维度 | 要求 |
|------|------|
| **语言风格** | 书面化但不枯燥，严谨但不死板 |
| **目标读者** | 年级前10%-15%的高中生，有思考能力 |
| **核心原则** | 生动、精准、有信息量 |

### 6.2 三大红线

#### 红线一：去口语化

禁止过度口语化的表达，保持学术文本的严谨性：

| ❌ 禁止 | ✅ 正确 |
|---------|--------|
| "说白了就是……" | "换言之，……" |
| "大家都知道……" | 直接陈述事实 |
| "咱们来看看……" | "分析如下：……" |
| "这玩意儿……" | 使用正式术语 |
| "你看这个……" | "观察以下……" |
| "其实很简单……" | 删除，直接给出分析 |

**底线**：可以适度口语化（如"你会发现……""注意到……"），但不能用聊天语气写教学内容。

#### 红线二：去AI化

禁止以下典型的AI生成文本模式：

| ❌ AI典型模式 | ✅ 替代方案 |
|--------------|------------|
| "让我们深入探讨……" | 直接进入主题 |
| "众所周知……" | 直接陈述，或给出出处 |
| "不难发现……" / "显而易见……" | 给出推理过程 |
| "这启示我们……" / "这告诉我们……" | 直接写结论 |
| "值得注意的是……" | 直接写出值得注意的内容 |
| "从某种意义上说……" | 删除或明确指出"哪种意义" |
| "总而言之……" / "综上所述……" | 用具体结论替代空泛总结 |
| 三连排比（"……的智慧，……的力量，……的魅力"） | 精简为一个有力表述 |
| "不仅……更……甚至……"递进三连 | 最多两层递进 |
| 每段以"首先/其次/最后"开头 | 用内在逻辑衔接，或用更自然的过渡 |
| 泛化赞美（"展现了物理学的深邃之美"） | 具体说明"美在哪里" |
| 空洞哲理（"知识的海洋浩瀚无垠"） | 删除，节省读者时间 |

**AI内容自检清单**：
- [ ] 是否存在"让我们""众所周知""不难发现"？→ 删除或替换
- [ ] 是否存在三连排比或递进？→ 精简
- [ ] 是否存在没有信息量的空泛赞美？→ 删除或替换为具体事实
- [ ] 是否每段都有实质内容（而非过渡废话）？→ 修剪

#### 红线三：去空洞化

每个段落必须有**实质信息增量**。禁止用修辞填充字数：

| ❌ 空洞 | ✅ 有信息量 |
|---------|------------|
| "牛顿三大定律是力学的基石" | "牛顿第二定律 $F=ma$ 将力与运动的关系定量化，是解题的起点" |
| "数学是一门优美的学科" | "导数的几何意义是切线斜率，这使得'变化率'这个抽象概念变得可视" |
| "掌握这个方法对学习大有裨益" | "使用此方法，3步内可完成原本需要5步的计算" |

**信息量自检**：删掉这句话后，读者是否损失了信息？如果不会，删掉。

### 6.3 学科差异说明

| 学科 | 风格微调 |
|------|---------|
| 理科（物/数/化/生） | 公式推导必须完整，每步有依据；逻辑链条可追溯 |
| 语文 | 引用原文必须准确标注出处；分析要有文本证据 |
| 英语 | 例句必须真实语料或高考真题；避免编造的"地道表达" |

| # | 检查项 | 评分标准 |

| **跨学科编码** | 节点在不同学科间的关联标识，格式如"P05↔K12↔C08"（P=物理，K=数学，C=化学） | §5.5 |
