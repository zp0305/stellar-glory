# 星耀高中教育平台 — 学习路径内容规划

> 定位：定义学习路径在网站上的组织方式、呈现结构和内容边界
> 核心原则：路径分散到学科指南和各专题里，不设独立一级入口；竞赛路径只放竞赛专区
> 模板策略：理科（物/数/化/生）一套模板，文科（语/英）一套模板

---

## 一、学习路径 vs 学科指南：边界划分

| | 学科指南 `/xxx/guide` | 学习路径 `/xxx/paths` |
|:---|:---|:---|
| **性质** | 方法论（静态的） | 时间线（动态的） |
| **核心问题** | 这门课怎么学、为什么学 | 什么时间点做什么 |
| **内容类型** | 学科本质、思维方法、学习技巧 | 阶段目标、模块顺序、进度节奏 |
| **示例** | "物理建模思维的5个步骤" | "高一上学期第5-8周：掌握牛顿第二定律应用" |
| **更新频率** | 低（学科方法相对稳定） | 中（每学期可能有调整） |

### 内容归属总表

| 内容 | 归属模块 | 网站位置 |
|:---|:---|:---|
| 初三暑假过渡方案 | 初高中衔接规划 | `/xxx/paths` → 暑假阶段 |
| 各科衔接重点清单 | 初高中衔接规划 | `/xxx/paths` → 暑假阶段 |
| 高一至高三各学期学习建议 | **本文件定义** | `/xxx/paths` → 各学期阶段 |
| 容错与调整方案 | 初高中衔接规划 | `/xxx/paths` → 容错板块 |
| 竞赛备考路径 | 竞赛规划 | `/competition/xxx`（仅竞赛专区） |
| 强基备考路径 | 强基规划 | `/xxx/senior` → 强基板块 |
| 学科学习方法 | 学习方法体系（v8.0模块01） | `/xxx/guide` 学科指南 |
| 三年整体规划 | 学习目标与三年规划（v8.0模块01） | `/xxx/guide` 学科指南 |
| 高考复习策略 | 高考专项内容规划 | `/xxx/senior` 高三专题 |

> **关键决策**：竞赛路径只在竞赛专区出现，不在学科学习路径页出现，避免对不走竞赛路线的学生产生干扰。

## 二、时间线结构（统一框架）

六科学习路径页采用统一的时间线框架，每科内容独立填充。

```
/xxx/paths（xxx = physics/math/chemistry/biology/chinese/english）
│

├── 📅 高一上学期（奠基期）
│   ├── 本学期目标（2-3个核心目标）
│   ├── 模块学习顺序（按教材章节，关联到星耀模型/套路）
│   ├── 星耀使用建议（推荐先学哪些模型、做哪些题）
│   └── 学期末自检（自检清单，链接到快速检测）
│
├── 📅 高一下学期（拓展期）
│   ├── ...
│
├── 📅 高二上学期（深化期）
│   ├── ...
│
├── 📅 高二下学期（突破期）
│   ├── ...
│
├── 📅 高三（冲刺期）
│   ├── 一轮复习建议（按知识模块）
│   ├── 二轮专题建议（按题型/方法）
│   ├── 三轮模考建议
│   └── → 高考专项详情（链接到 /xxx/senior）

    ├── 6种典型偏差及调整策略（来自初高中衔接规划）
```

### 各阶段命名与定位

| 阶段 | 时间 | 定位 | 核心任务 |
|:---|:---|:---|:---|
| 暑假准备期 | 初三毕业→8月底 | 补短板+建方法 | 诊断薄弱点、衔接断层知识、建立高中学习方法 |
| 高一上·奠基 | 9月→1月 | 跟上节奏+打基础 | 适应高中节奏，完成必修一核心模块 |
| 高一下·拓展 | 2月→7月 | 稳步推进+开始思考 | 完成必修二，了解学科全貌，思考竞赛/强基意向 |
| 高二上·深化 | 9月→1月 | 重点突破+选科锁定 | 选择性必修重点模块，确定是否走竞赛/强基 |
| 高二下·突破 | 2月→7月 | 能力跃迁+综合训练 | L2思维运用训练，开始接触压轴题 |
| 高三·冲刺 | 9月→6月 | 系统复习+应试 | 一轮全面→二轮专题→三轮模考 |

## 三、路径节点与网站内容的关联

> 学习路径不是孤立的文字建议，而是**串联起网站所有内容模块的导航骨架**。

### 3.1 节点关联规则

每个学习路径中的"模块学习节点"应关联到以下网站内容：

| 关联类型 | 链接目标 | 说明 |
|:---|:---|:---|
| 模型学习 | `/xxx/models/{modelId}` | 推荐学习哪个模型 |
| 套路训练 | `/xxx/strategies/{strategyId}` | 推荐掌握哪个套路 |
| 练习检测 | `/xxx/practice/quiz?mode=...` | 推荐做哪些题（按知识点/套路/难度） |
| 快速检测 | `/xxx/practice/quiz?mode=level` | 阶段结束时的自检 |
| 学科故事 | `/xxx/vision/{storyId}` | 配合学习的拓展阅读 |
| 高三专题 | `/xxx/senior` | 高三阶段的专项内容 |

### 3.2 节点数据结构

```typescript
interface PathNode {
  id: string;                    // 节点ID，如 "physics-g1-m3"
  title: string;                 // 节点标题，如 "牛顿第二定律应用"
  weekRange: string;             // 建议周次，如 "第5-8周"
  description: string;           // 学习建议（2-3句话）
  
  // 关联内容（可多个）
  models?: string[];             // 关联模型ID
  strategies?: string[];         // 关联套路ID
  practices?: PracticeLink[];    // 关联练习
  visionStories?: string[];      // 关联学科故事ID
  
  // 检测
  checkpoint?: {
    description: string;         // 自检说明
    practiceLink: string;        // 检测练习链接
  };
}

interface PracticeLink {
  mode: 'knowledge' | 'strategy' | 'level';
  params: Record<string, string>;
  label: string;                 // 显示文字，如 "匀变速运动·基础练习"
}

interface SemesterPath {
  id: string;
  semester: string;              // "g1s1" | "g1s2" | "g2s1" | "g2s2" | "g3"
  title: string;                 // "高一上学期·奠基期"
  goals: string[];               // 本学期目标
  nodes: PathNode[];             // 学习节点列表
  checkpoint: string;            // 学期末自检说明
}
```

### 3.3 关联示例

以物理高一上学期的"运动学"节点为例：

```
路径节点：匀变速直线运动
  周次：第3-6周
  建议：掌握匀变速直线运动的五大公式，能处理自由落体和竖直上抛问题
  关联模型：→ 匀变速直线运动模型（/physics/models/uniform-acceleration）
  关联套路：→ 运动学基本公式套路（/physics/strategies/kinematics-formulas）
           → 追及相遇问题套路（/physics/strategies/pursuit-meeting）
  推荐练习：→ 运动学·B级练习（基础巩固）
  拓展阅读：→ 伽利略与自由落体（/physics/vision/galileo-freefall）
  阶段检测：→ 运动学综合检测（链接到快速检测）
```

- 理科知识模块界限清晰，按教材章节组织

## 五、文科模板（语/英）

### 5.1 模板特点

- 文科不像理科有明确的"章节→模型"对应关系，更多是能力线的递进
- 时间线不完全按教材走，而是按**能力发展阶段**组织
- 每日积累比集中突破更重要（词汇、阅读、写作都是长期习惯）
- 与理科模板共享时间线框架，但每个阶段的内容组织方式不同

### 5.2 文科阶段的命名差异

| 理科阶段名 | 文科替代名 | 差异原因 |
|:---|:---|:---|
| 模块学习顺序 | **能力训练线** | 文科不按章节，按能力（阅读/写作/积累） |
| 关联模型 | **关联专项训练** | 文科没有"模型"概念，对应的是题型训练 |
| 关联套路 | **关联答题方法** | 对应文科的答题框架和写作模板 |

### 6.2 页面交互设计建议

| 交互 | 说明 |
|:---|:---|
| **时间线视图** | 默认展示，垂直时间线，每个学期一个节点 |
| **展开/折叠** | 点击学期节点展开具体周次内容 |
| **进度标记** | 用户完成某节点后可标记（需登录态，暂缓） |
| **内容跳转** | 每个节点中的"星耀关联"可直接跳转到对应模块 |
| **当前学期高亮** | 根据当前日期自动高亮当前学期 |
| **文科特殊处理** | 语文/英语在时间线基础上，额外展示"持续积累"条目（贯穿全学期） |

### 6.3 暑假内容和容错内容的复用

暑假阶段和容错板块的内容来自《初高中衔接专题内容规划》，在数据层面有两种方案：

| 方案 | 说明 | 优缺点 |
|:---|:---|:---|
| **A. 数据引用** | paths.ts 中引用衔接规划的数据，前端统一渲染 | 修改一处，六科同步更新；但数据耦合 |
| **B. 数据复制** | 每科的 paths.ts 中各存一份（可按科目裁剪） | 数据独立，可按科目定制；但更新成本高 |

建议采用**方案B**，因为六科的衔接重点不同（如物理强调矢量，语文强调文言文），需要独立定制。
